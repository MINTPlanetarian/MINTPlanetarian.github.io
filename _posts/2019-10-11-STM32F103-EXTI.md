##一、外部中断简介##
STM32的中断可以分为内部中断和外部中断，外部中断由外部中断控制器（EXTI）控制。STM32F10X的外部中断控制器包含多达20个用于产生中断请求的边沿检测器。EXTI的每根输入线都可单独进行配置，以选择类型（中断或事件）和相应的触发事件（上升沿触发、下降沿触发或边沿触发），还可独立地被屏蔽。

从框图中可以看到，从输入线开始有两条线路的分化，其中一条主管事件（对应图中“脉冲发生器”一路），另一条主管中断（对应图中“至NVIC中断控制器”一路）。我主要学习的是中断，其分为以下几个重要部分：<p>
1. 上升/下降沿触发选择寄存器：这两个寄存器主要是控制触发事件是上升沿触发、下降沿触发还是上升、下降沿都能触发。而上升/下降沿指的是电平由0到1的过程或由1到0的过程，也即电平跳变。当发生指定的跳变类型时，边沿检测电路将输出有效信号1，否则输出0。<p>
2. 软件中断事件寄存器：如果说上升/下降沿触发选择寄存器代表的是硬件方面的中断输入识别的话，该寄存器代表就是软件方面对外部中断的控制。从“与门”我们可以看到只有硬件和软件都允许中断启动时，中断才能继续运行，这时“与门”发送有效信号1。<p>
3. 请求挂起寄存器：用于储存前所述的“与门”输出的信号。<p>
4. 中断屏蔽寄存器：用于决定中断是否被屏蔽。也就是说如果该寄存器被设定为对中断进行屏蔽，那么无论上述过程为如何，中断都不会正常启动。<p>在进行完上述步骤后，请求挂起寄存器内的信号将被送到NVIC内，以实现系统对中断的控制。

##二、外部中断配置##
从资料中我们可以得到如下信息：EXTI只有0~15是对应外部IO口的输入中断的。但我们又知道STM32F103不只有16个IO口，所以STM32只能通过管脚复用的方式，来使GPIOx.0-GPIOx.15（x=A,B,C,D,E,F,G）分别对应EXTI0-EXTI15，之后在通过配置AFIO来确定每个EXIT0具体对应哪一个GPIO。（更详细的关于AFIO的介绍请参考：[STM32F103学习笔记-AFIO相关](https://blog.csdn.net/qq_43174086/article/details/97028280)）<p>
故，外部中断的配置过程如下：<p>
1. 使能IO口时钟，按需要配置IO口模式；<p>
2. 开启AFIO时钟，设置IO口与中断线的映射关系；<p>
3. 配置中断分组（NVIC），使能中断；<p>
4. 初始化EXTI，选择触发方式；<p>
5. 编写EXTI中断服务函数。

外部中断配置示例：

	//使能GPIO的过程略
	
	RCC_APB2PeriphClockCmd(RCC_APB2Periph_AFIO,ENABLE); //开启AFIO时钟
	
	void GPIO_EXTILineConfig(uint8_t GPIO_PortSource, uint8_t GPIO_PinSource); //将GPIO和EXTI建立连接（映射）
	
	//下为NVIC的配置过程
	
	NVIC_InitTypeDef NVIC_InitStructure;
	NVIC_InitStructure.NVIC_IRQChannel = EXTIx_IRQn; //EXTIx 中断通道
	NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority=m; //抢占优先级	
	NVIC_InitStructure.NVIC_IRQChannelSubPriority = (5-m); //响应优先级	
	NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE; //IRQ通道使能	
	NVIC_Init(&NVIC_InitStructure); //根据指定的参数初始化NVIC寄存器
	
	//下为初始化EXTI，选择触发方式
	
	EXTI_InitTypeDef EXTI_InitStructure;
	EXTI_InitStructure.EXTI_Line=EXTI_Linex; //配置EXTI中断线
	EXTI_InitStructure.EXTI_Mode=EXTI_Mode_Interrupt; //配置EXTI模式为中断
	EXTI_InitStructure.EXTI_Trigger=EXTI_Trigger; //配置EXTI触发方式
	EXTI_InitStructure.EXTI_LineCmd=ENABLE; //使能EXTI中断线
	EXTI_Init(&EXTI_InitStruct); //根据指定的参数初始化EXTI寄存器
	
	//下为编写EXTI中断服务函数
	
	void EXTIx_IQRHandler(void)
	{
		if(EXTI_GetITStatus(EXTI_Linex) == 1) //读取中断标志位，确定中断真的发生
		{
		//此处编写中断函数
		}
		EXTI_ClearITPendingBit(EXTI_Linex); //清除中断标志位
	}

特别注意：以上示例中的x（包括EXTIx_IQRn、EXTI_Linex）代表了16根中断线的编号，所以取值范围为0~15。

##三、总结##
中断作为STM32的一个重要部分，我花了很久的时间来琢磨它，从它的功能到它的配置，再到和它有关的AFIO的相关知识。学STM32到现在，发现初始化与配置指定功能函数是非常有套路且流程化的，但真正编写中断函数和设计及实现功能却十分麻烦。当然，配置与初始化作为基础是不能丢的。日后，将在配置的基础上进行些项目的设计，以把学到的东西应用起来。
